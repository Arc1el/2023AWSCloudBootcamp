# Section 1. 요금 기본 사항
- AWS의 3가지 기본적인 비용 동인
  - 컴퓨팅
    - 시간/초당 청구
    - 인스턴스 유형에 따라 다름
  - 스토리지
    - 일반적으로 GB당 청구
  - 데이터 전송
    - 아웃바운드 요금은 집계하여 청구
    - 인바운드는 무료(일부 예외). 동일 AWS리전 내 다른 AWS서비스간 무료
    - 일반적으로 GB당 청구
- AWS에서의 요금 지불 방법
  - 각 서비스에서 실제로 필요한 리소스의 양에 대해 정확히 지불
  - 사용한 만큼만 지불
    - 고액의 초기 지출 없이 사용한 서비스에 대해서만 비용을 지불.
    - 가변 비용을 낮출 수 있으므로, 고가의 인프라 구축에 귀중한 리소스 투자 필요 없음.
    - 변화하는 비즈니스 요구에 빠르게 적응하고 필요한만큼만 사용료를 지불하여 혁신과 발명에 집중
    - 모든AWS 서비스는 온디맨드로 제공되며, 장기계약을 맺지 않아도 되고, 복잡한 라이선스에 의존할 필요가 없음.
  - 예약시 더 적은 비용 지불
    - 특정 서비스의 경우 예약 용량에 투자 할 수 있음.
    - AURI(전체 선결제 예약 인스턴스)
    - PURI(부분 선결제 예약 인스턴스)
    - NURI(선결제 없는 예약 인스턴스)
  - 사용량이 많을수록 비용 절감.
    - 사용량이 증가함에 따라 비용절감도 증대
    - 계층화된 요금제를 사용하는 경우 사용량이 늘수록 GB당 지불하는 비용이 감소.
    - 요구사항에 따라 여러 스토리지 서비스를 사용하여 스토리지 비용을 절감 할 수 있음.
  - AWS성장에 따라 비용 절감액도 증가
    - AWS는 규모의 경제를 통해 절감되는 비용을 고객에게 돌려줌.
    - 2006년 이후로 2019년 9월기준 요금을 75회 인하
    - 향후 더 우수한 성능ㅇ르 제공하는 리소스가 나오면 현재 리소스가 추가비용 없이 대체됨.
- 사용자 지정 요금
  - 사용자 지정 요금을 통해 다양한 요구사항을 충족 가능.
  - 고유한 요구사항이 있는 대용량 프로젝트에 사용할 수 있음.
- AWS 프리티어
  - AWS는 신규 고객에게 1년간 AWS플랫폼, 특정 제품 및 서비스를 무료체험 기회를 제공
- 무료서비스
  - Amazon VPC
    - 논리적으로 격리된 공간을 프로비저닝하고 고객이 정의한 가상네트워크를 사용하여 리소스를 시작할 수 있음.
  - Elastic Beanstalk
    - AWS클라우드에서 애플리케이션을 신속하게 배포하고 관리
  - Auto Scaling
    - 정의한 조건에 따라 리소스를 자동으로 추가하거나 제거
  - AWS CloudFormation
    - 개발자 및 시스템 관리자가 AWS리소스 모음을 쉽게 생성하고, 순서에 따라 예측 가능한 방식으로 프로비저닝 하도록 지원.
  - AWS Identitiy and Access Management(IAM)
    - AWS서비스 및 리소스에 대한 사용자의 액세스를 제어
> 인바운드 데이터전송, 동일한 AWS리전 내 서비스 간 데이터 전송에는 요금 부과되지 않음<br>
> 실제 사용한 만큼만 지불<br>
> 언제든지 시작하고 중지<br>
> 장기계약은 필요 없음<br>
> 이루 서비스는 무료이지만 서비스에서 프로비저닝 하는 다른 AWS 서비스는 무료가 아닐 수 있음
